# Demo design system

Purpose: a high-density scientific developer console for interview demos and technical reviews. Tone: auditable instrumentation, not consumer chat. The memorable interaction is the evidence report rail: status, confidence, replay interval, modality, speaker/page, missing evidence, and tool trace appear in one reading path.

Tokens: deep navy `#07101a`, panel `#0c1925`, evidence cyan `#55e6d5`, uncertainty amber `#ffc76b`, abstention red `#ff7f86`, body ink `#edf7ff`. Components use 10–18 px radii, one-pixel slate borders, and visible focus rings. Color is never the only status cue. The page supports 375–1440 px layouts and `prefers-reduced-motion`.
